# Zac Dagostino Portfolio

A playful neo-brutalist portfolio for Zac Dagostino, built with Next.js App Router, React, TypeScript and Tailwind CSS.

## Run Locally

```bash
npm install
npm run dev
```

Open `http://localhost:3000`.

## Quality Commands

```bash
npm run lint
npm run typecheck
npm run build
```

## Editing Projects

Project content lives in [data/site.ts](/workspaces/web-dev-website/data/site.ts).

Update the `projects` array to change:

- project title
- category
- real/concept status
- problem
- weird idea
- practical build work
- intended result
- accent colour

The expanding project grid is rendered by [components/ProjectGrid.tsx](/workspaces/web-dev-website/components/ProjectGrid.tsx).

## Editing Colours

Core brand colours are defined in [tailwind.config.ts](/workspaces/web-dev-website/tailwind.config.ts).

The main palette is:

- `paper`
- `ink`
- `pink`
- `acid`
- `yellow`
- `cyan`
- `coral`
- `lilac`

Global defaults and CSS-only effects are in [app/globals.css](/workspaces/web-dev-website/app/globals.css).

## Editing The Hero

The cleaner static hero is rendered by [components/Hero.tsx](/workspaces/web-dev-website/components/Hero.tsx).

Useful places to edit:

- Identity, location and supporting copy come from [data/site.ts](/workspaces/web-dev-website/data/site.ts).
- The static browser-window graphic is inside the right-side hero panel.
- The `WORK` module is the final acid-green grid block in the hero.

## Contact Form Settings

The form is in [components/ContactForm.tsx](/workspaces/web-dev-website/components/ContactForm.tsx).

Project type and budget options come from [data/site.ts](/workspaces/web-dev-website/data/site.ts).

The current form validates on submit and shows the requested success message locally. To connect it to a real service, replace the `handleSubmit` success block with a request to your preferred form endpoint or server action, then only call `setSubmitted(true)` after the request succeeds.
